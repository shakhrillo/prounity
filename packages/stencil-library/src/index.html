PU Avatar

Avatars are circular components that usually wrap an image or icon. They can be used to represent a person or an object.

Avatars can be used by themselves or inside of any element. If placed inside of an ion-chip or ion-item, the avatar will resize to fit the parent component. To position an avatar on the left or right side of an item, set the slot to start or end, respectively.

import {pu-avatar} from '../../

function Example() {
return (
<>
<IonAvatar>
<img alt="Silhouette of a person's head" src="https://ionicframework.com/docs/img/demos/avatar.svg" />
</IonAvatar>
</>
);
}
export default Example;

PU Badge

Badges are inline block elements that usually appear near another element. Typically they contain a number or other characters. They can be used as a notification that there are additional items associated with an element and indicate how many items there are. Badges are hidden if no content is passed in.

PU Card

Cards are containers that display content such as text, images, buttons, and lists. A card can be a single component, but is often made up of a header, title, subtitle, and content. Cards are broken up into several components to accommodate this structure: card header, card title, card subtitle, and card content.

import {pu-card} from '@pu/react'

function Example() {
return (
<>
<pu-card
title="Card Title"
subtitle="Card Subtitle"
content="Here is Card Content">
</pu-card >
</>
);
}

PU Input

PU Toast

The isOpen property on ion-toast allows developers to control the presentation state of the toast from their application state. This means when isOpen is set to true the toast will be presented and when isOpen is set to false the toast will be dismissed.

import {pu-toast} from '@pu/react'

function Example() {
return (
<>
<pu-toast time="3000" text="Toast Lorem ipsum dolor sit amet." >
</pu-toast>
</>
);
}
export default Example;
